## 💡 System Blueprint: The Three Ws

### 1. WHEN to use this tool?
*   **Continuous Integration Gates (CI):** Before merging a developer's raw shell script into a staging branch, run this tool to check for structural loopholes or hidden injection surfaces.
*   **Legacy Code Onboarding:** When transitioning an old, undocumented code stack over to a new team and you need rapid, precise, markdown architecture summaries instantly.
*   **Automating Code Review Cycles:** To offload mundane linting, safety auditing, and documentation writing from senior engineers over to a targeted, deterministic AI pipeline.

### 2. WHERE does it run?
*   **Local Terminals & Automated Runners:** Runs as a standalone Python utility on any developer machine or as a continuous validation worker thread within CI/CD pipelines (GitHub Actions, GitLab CI, Jenkins runners).

### 3. WHY use this over other solutions?
*   **Zero-Config Failure Resilience:** Most AI CLI automation utilities crash immediately with unhandled errors if your API network connection dropped or if a configuration key is missing. This tool auto-evaluates its environment and seamlessly slides into a zero-token simulation mode so execution pipelines never block your developers.
